ClinicDesk Management System

An Electronic Medical Records (EMR) and clinic automation platform built from the ground up to solve complex scheduling, clinical histories, and automated invoicing splits.

● 83 Test Suites Passing
● Dual LTR/RTL Arabic Support
● Stripe checkout & webhook active
🔐 Granular RBAC

Five specific user roles (Guest, Patient, Receptionist, Doctor, Admin) each with distinct dashboards and permissions validated by backend checks.

🩺 EMR Integration

Doctor daily queue management, clinical notes logger, vital signs aggregator, ICD-10 diagnoses databases, and print-friendly prescription managers.

💳 Automated Billing

Invoice automation triggered directly by consultations, split cash/Stripe tracking, automated partial refunds, and full accounting statistics.

Traditional Clinic Bottlenecks

Small to medium clinics in the regional MENA market rely heavily on manual paper-based processes. This introduces massive room for operational errors, client frustration, and lost revenue.

Paper-Based Medical Records

Lost history, messy handwriting, zero search capability. Doctors make decisions with incomplete medical records.

Chaos in Front Desk Coordination

Receptionists juggle phone bookings, walk-ins, and cash billing with no slot conflict warnings or status tracking.

Lost Income & No-Shows

Clinics take the hit when patients cancel late or fail to show. No automated reminder command is present in legacy models.

The ClinicDesk Solution

An integrated platform automating operations from onboarding to payment, wrapped in a beautiful localizable shell.

✔ Structured Electronic EMR

Securely stores allergies, vital stats, chronic conditions, and previous visits. Doctor can view a patient's complete history immediately.

✔ Dual-mode Appointment Workflow

Online scheduling checks slot availability dynamically, locks slot, and takes Stripe payment. Receptionist walk-ins bypass Stripe for cash billing.

✔ Automatic Invoice Splits & Refunds

Automatic invoice generated. Stripe cancels trigger refund rules: 80% back if self-cancelled by patient, 100% back if cancelled by staff.

✔ native LTR/RTL Arabic Support

Every form validation, navigation link, model representation, and layout switcher adapted for the local Arabic environment.

Tooling Pivot: Why We Upgraded

During initial planning, OpenSpec fell short in accuracy, leading the engineering team to adopt the BMAD structure.

✗ Limitations of OpenSpec
  • Inaccurate API specifications: Often hallucinated query variables and headers mismatching actual Django forms parameters.
  • Zero State Awareness: Ignored intermediate database state transitions (e.g. checked-in vs scheduled).
  • No Business Rules Context: Left out cancellation policy calculations, Stripe webhook payload layouts, and specific locale configs.
  • Poor Developer Handoffs: Lacked a precise step-by-step epic structure, causing code conflicts and implementation gaps.
✓ The BMAD Framework Advantage
B Business Focus

Establishes the exact product goals, priorities, and 14 missing feature sets in `prd.md` before coding.

M Model Definitions

Maps domain rules, relationship summaries, index schemas, and constraint specifications in `architecture.md`.

A Application Logic

Defines explicit route directories, endpoint parameters, and templates designs in `API_DOCUMENTATION.md`.

D Delivery Tracking

Slices implementation into structured backlog epics (`epic-01` to `epic-13`) verifying each phase with rigorous unit testing.

API & Endpoint Specifications

The backend exposes a clean JSON/AJAX interface alongside traditional HTML views to handle reactive UI updates.

Method Path Format Role Required Description
GET /appointments/slots/ JSON Patient / Staff Returns unbooked time intervals for doctor and date parameters.
POST /appointments/book/<slot_id>/ JSON Patient / Staff Locks time slot and creates a Stripe checkout redirect url.
POST /payments/webhook/ Plain System (Stripe) Processes successful payment updates and issues invoices.
GET /appointments/my/<app_id>/cancel/preflight/ JSON Patient Calculates exact refund percentages (80% vs 100%) before cancellation confirmation.
POST /notifications/<pk>/read/ JSON All Roles Sets unread notification flag to read for specific item.
GET /notifications/unread-count/ JSON All Roles Returns active unread counts for UI bell polling.
POST /payments/invoice/<pk>/pay/ Redirect Receptionist Records manual cash/POS split payment against issued billing invoice.

Technology Integration Stack

An architectural overview of the framework components driving the ClinicDesk portal.

🐍 Django 6.0

Acts as the core transactional framework. Handles models representation, secure session control, custom decorators, and middleware rules.

💳 Stripe Checkout API

Processes payment cards securely. Operates checkout callbacks, webhook transaction logging, and automated partial refund distributions.

🐬 MySQL / SQLite

Relational storage engines enforcing strict foreign keys, indexing profiles, and decimal precision integrity for billing cards.

🌍 gettext i18n Core

Standard locale utility translating messages catalogs, compiling RTL styling directives, and switching contexts instantly.
